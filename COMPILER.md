# PI 编译器：SKILL_META.md → SKILL.md

> SKILL_META.md 是迭代真源（完整设计文档），SKILL.md 是编译产物（给 AI 用的精简版）。
> LLM 不是 LLVM——不追求过度压缩 token，追求信息不失真+高信息密度。
> 本文件定义编译规则，后续对精简策略的讨论和优化基于此文件进行。
> 属于四阶段流水线的 Phase 2。完整流水线：iterate.md(P1迭代) → COMPILER.md(P2编译) → DISTRIBUTE.md(P3分发) → TRANSLATE.md(P4翻译)

## 架构

```
SKILL_META.md  ← 迭代真源（完整版·给人看·~1807行）
      │
      ├──→ COMPILER.md（标准规则）     → skills/pi/SKILL.md         (原版精简·~700行)
      └──→ COMPILER.md（渐进Core规则） → skills/pi-progressive/SKILL.md (Core·~200行)
                                      → skills/pi-progressive/references/*.md (按角色扩展)
```

## 编译原则

- **行为指令** = 必须保留：祈使句、触发条件、格式模板、表格中的"动效"列
- **认知策略** = 必须保留：MBTI 认知功能栈是处理优先级参数集（Ni→Te = 先洞察再执行），认知阵/认知原型/认知流管线均为 AI 的信息处理指令
- **设计意图** = 仅在 META：古典引用、思想源、"本质"说明、"为什么"段落、示例
- **边界注解** = 选择性保留：影响 AI 判断"何时用/不用"的保留，纯解释性的删
- **格式化输出模板** = 必须保留：用户需要看到 PI 生效的信号
- **信息不失真** > 压缩 token：LLM 不是字节码解释器，不追求过度压缩

## 全局剥离规则（适用于所有章节）

| 剥离项 | 理由 | 示例 |
|--------|------|------|
| 章节开头古典引语 (`> **...** ——《...》`) | 不改变 AI 行为 | `> **道生一...** ——《道德经》` |
| 表格中 `经典` 列 | 装饰性，AI 不因此改变决策 | 五敕令表、五律表、六阶战势表的经典列 |
| 表格中 `思想源` 列 | 哲学归属不改变处理策略（认知栈已承载） | 五略表、编程四令表 |
| `> 示例：...` 代码块 | 保留格式模板，删示例实例 | 明链/明证/明树/明心/明约的示例块 |
| `**本质**：...` 解释段落 | 给人看的设计意图 | 共振五式各式的"本质"段 |
| `**为什么需要...？**` 段落 | 设计 rationale | 共振五式总论的"为什么" |
| `**用户干预点**` / `**与反模式联动**` 段落 | 分析性文字，非行为指令 | 明链/明证的联动说明 |
| `**人机协议**` 详细说明（保留行为规则） | 4条规则压为1-2行 | 明树人机协议 |
| 尾部哲理签名 | 纯装饰 | `🌊 兵无常势...` |

## 章节标题格式

所有章节标题从 `## 第X章 · Y — Z` 简化为 `## X. Z`。删除古典元素符号（道/法/术/器/势/灵/和/人），仅保留功能名。

## 逐章详细方案

### frontmatter 编译（description 触发优化）

SKILL_META.md 的 description 已是纯触发关键词（≤400字符），编译时**原样保留或进一步压缩至≤300字符**（删冗余关键词）。

**编译规则**：

| 序 | 规则 | 理由 |
|---|------|------|
| 一 | **仅保留用户搜索词** | 用户输入"调试""写代码"而非"九令洞鉴""反模式十戒" |
| 二 | **删除所有内部概念** | 敕令名/机制名/模块名是正文内容，不是触发关键词 |
| 三 | **保留行为信号** | 失败次数/打转/言退/再试试/算了 = AI 判断何时升级的信号 |
| 四 | **字符数 ≤ 400（META）/ ≤ 300（编译产物）** | 短 description = 快匹配 = 高触发率 |
| 五 | **格式固定** | `{引擎名}。触发：{关键词/分隔}，或{行为信号}` |

**编译前校验**：编译前先检查 SKILL_META.md 的 description 是否符合纯触发词规范。若发现内部概念（敕令名/机制名/模块名/流程描述），在编译前先修正 SKILL_META.md。

**编译模板**（从 SKILL_META.md 触发词原样保留，可微调压缩）：

```
中文：PI 智行合一。触发：编程/开发/fleet/代码/架构/API/调试/bug/报错/测试/编译/compile/test/git/make/发布/验证/产品/需求/运营/增长/创意/设计/协作/团队/沟通/交互/陪伴/情感，或失败2+次/打转/言退/再试试/换个参数/算了
白话：已废弃
英文：PI Cognitive AI. Trigger: coding/development/fleet/architecture/API/debugging/bug/error/testing/compile/test/git/make/release/verify/product/requirements/ops/growth/creative/design/collaboration/team/communication/interaction/support, or 2+ failures/looping/giving-up/retry/nevermind
```

**关键词选取原则**：每个关键词必须是用户可能输入的搜索词。用户不会输入的词（五敕令/反模式十戒/认知原型/战势/九令洞鉴/经验三域/三司会审/战域卡等）一律删除。description 的唯一使命是**触发**，不是**描述**。

### Core层编译规则（~200行目标）

**Core原则**：每一行都必须直接驱动至少1个评测指标。达不到此标准的内容 → Advanced层（references/）。

**通用能力审计（防 Benchmark Overfitting）**：
- Core 每条规则必须**同时满足**：①驱动 ≥1 个评测指标 ②在真实编程/调试/审查场景中同样有效
- 审计信号：某条规则在评测中提分但真实场景中被用户跳过/忽略 → 降级到 references/
- 原则：**通用能力 > 评测得分**。评测是能力的度量，不是能力的目标。针对特定评测场景的 hack 禁止进入 Core

- 致人术三式：§3.2 定义 + 调试七步·扩圈/交付六令多处引用 → 定义保持§3.2，引用处用交叉引用（如"致人术·同类排查§3.2"）

### 前缀压缩去重执行指引（KV Cache 友好·编译时强制）

> PI SKILL 作为系统提示是稳定前缀（KV Cache 友好），但内部跨章节重复增加首次计算成本和 token 消耗。
> 编译器（Opus 4.6）在编译时必须主动执行去重，不能等人工标记。

**编译时去重流程**（每次编译必须执行）：

| 步骤 | 动效 | 操作 |
|------|------|------|
| 一·扫描 | 识别重复 | 编译完成后，全文扫描编译产物，找出**语义相同但位置不同**的内容块（同一概念/规则/格式在多处出现） |
| 二·定源 | 确定首发 | 每个重复概念选择**首次出现位置**作为定义源（canonical definition） |
| 三·替引 | 交叉引用 | 后续出现位置替换为 `(§X.Y)` 交叉引用，保留引用意图但删除重复全文 |
| 四·验损 | 校验无损 | 替换后确认：①引用目标确实存在 ②删除的内容不包含首发位置没有的增量信息 ③AI 仅凭交叉引用能还原完整语义 |

**去重判定标准**：
- ✅ 应去重：同一规则/格式/表格在多处**全文重复**（如验证矩阵、隐患40%标准）
- ✅ 应去重：同一行为指令用**不同措辞重复表达**（如"改必验证"在五敕令、调试七步、交付六令各说一遍）
- ❌ 不去重：同一概念在不同**上下文中有增量信息**（如致人术在§3.2定义详细用法，在调试七步·扩圈中补充调试专用的检查清单）
- ❌ 不去重：**核心铁律的重复强调**（如⚡PI-01~05 在多处出现是刻意的，确保 LLM 注意力不衰减）

**token 预算**：编译产物中可消除的冗余 ≤ 总 token 的 15-20%。去重后标准版目标 ~700行，Core 版 ~200行。超此范围说明可能误删了非重复内容。
