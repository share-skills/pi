# PI 编译器：SKILL_META.md → SKILL.md

> SKILL_META.md 是迭代真源（完整设计文档），SKILL.md 是编译产物（给 AI 用的精简版）。
> LLM 不是 LLVM——不追求过度压缩 token，追求信息不失真+高信息密度。
> 本文件定义编译规则，后续对精简策略的讨论和优化基于此文件进行。
> 属于四阶段流水线的 Phase 2。完整流水线：iterate.md(P1迭代) → COMPILER.md(P2编译) → DISTRIBUTE.md(P3分发) → TRANSLATE.md(P4翻译)

## 架构

```
SKILL_META.md  ← 迭代真源（完整版·给人看·~1170行）
      │
      ├──→ COMPILER.md      → SKILL.md      (原版精简·给 AI 用·~807行)
      │
      └──→ COMPILER_LITE.md → SKILL_LITE.md (白话版·功能等价·零术语)
```

## 编译原则

- **行为指令** = 必须保留：祈使句、触发条件、格式模板、表格中的"动效"列
- **认知策略** = 必须保留：MBTI 认知功能栈是处理优先级参数集（Ni→Te = 先洞察再执行），认知阵/认知原型/认知流管线均为 AI 的信息处理指令
- **设计意图** = 仅在 META：古典引用、思想源、"本质"说明、"为什么"段落、示例
- **边界注解** = 选择性保留：影响 AI 判断"何时用/不用"的保留，纯解释性的删
- **格式化输出模板** = 必须保留：用户需要看到 PI 生效的信号
- **信息不失真** > 压缩 token：LLM 不是字节码解释器，不追求过度压缩

## 全局剥离规则（适用于所有章节）

| 剥离项 | 理由 | 示例 |
|--------|------|------|
| 章节开头古典引语 (`> **...** ——《...》`) | 不改变 AI 行为 | `> **道生一...** ——《道德经》` |
| 表格中 `经典` 列 | 装饰性，AI 不因此改变决策 | 五敕令表、五律表、六阶战势表的经典列 |
| 表格中 `思想源` 列 | 哲学归属不改变处理策略（认知栈已承载） | 五略表、编程四令表 |
| `> 示例：...` 代码块 | 保留格式模板，删示例实例 | 明链/明证/明树/明心/明约的示例块 |
| `**本质**：...` 解释段落 | 给人看的设计意图 | 共振五式各式的"本质"段 |
| `**为什么需要...？**` 段落 | 设计 rationale | 共振五式总论的"为什么" |
| `**用户干预点**` / `**与反模式联动**` 段落 | 分析性文字，非行为指令 | 明链/明证的联动说明 |
| `**人机协议**` 详细说明（保留行为规则） | 4条规则压为1-2行 | 明树人机协议 |
| 尾部哲理签名 | 纯装饰 | `🌊 兵无常势...` |

## 章节标题格式

所有章节标题从 `## 第X章 · Y — Z` 简化为 `## X. Z`。删除古典元素符号（道/法/术/器/势/灵/和/人），仅保留功能名。

## 逐章详细方案

### frontmatter 编译（description 触发优化）

SKILL_META.md 的 description 是完整索引（给人看），编译时**压缩为纯触发关键词版本**（给 AI 匹配用）。

**编译规则**：

| 序 | 规则 | 理由 |
|---|------|------|
| 一 | **仅保留用户搜索词** | 用户输入"调试""写代码"而非"九令洞鉴""反模式十戒" |
| 二 | **删除所有内部概念** | 敕令名/机制名/模块名是正文内容，不是触发关键词 |
| 三 | **保留行为信号** | 失败次数/打转/言退 = AI 判断何时升级的信号 |
| 四 | **字符数 ≤ 300** | 短 description = 快匹配 = 高触发率 |
| 五 | **格式固定** | `{引擎名}。触发：{关键词/分隔}，或{行为信号}` |

**编译模板**：

```
中文：PI 智行合一。触发：编程/开发/fleet/代码/架构/API/调试/bug/报错/测试/编译/compile/test/git/make/发布/验证/产品/需求/运营/增长/创意/设计/协作/团队/沟通/交互/陪伴/情感，或失败2+次/打转/言退/再试试/换个参数/算了
白话：同上 + 开头加（白话版）
英文：PI Cognitive AI. Trigger: coding/development/fleet/architecture/API/debugging/bug/error/testing/compile/test/git/make/release/verify/product/requirements/ops/growth/creative/design/collaboration/team/communication/interaction/support, or 2+ failures/looping/giving-up/retry/nevermind
```

**关键词选取原则**：每个关键词必须是用户可能输入的搜索词。用户不会输入的词（五敕令/反模式十戒/认知原型/战势/九令洞鉴等）一律删除。

## 编译校验清单

编译完成后逐条检查：

1. 行为指令无丢失（逐条比对）
2. 输出格式模板完整（场景公示、触发通知、PI·战报、肃阵模板、天行终极模板、情报输出格式、场景桥接格式、方案比选格式、明链/明证/明树/明心/明约、已试策略簿、快照、善始善终、恢复、三省、MMR commit）
3. 触发条件完整
4. description 索引的概念在精简版中均有定义
5. emoji 情绪价值适当
6. `§` 引用零悬空：精简后所有 `§X.Y` 交叉引用均有对应段落，且 Anchor 名称与最新章节一致
7. 混合表格裁剪一致：凡标记"删经典/删思想源"的表格需严格按列或按单元格裁剪，禁止出现半裁剪的混合列

如发现丢失：回溯 SKILL_META.md 确认意图，补回 SKILL.md
