## 阶段详解

---

### P1 · 迭代 — 内容演进

```
📄 iterate.md (审计协议)
┌─────────────────────────────────────────────┐
│                                             │
│   输入: SKILL_META.md (唯一迭代真源)          │
│   产出: 更新后的 SKILL_META.md               │
│   工具: 五维十四检 + 九大逻辑链               │
│                                             │
│   ┌─────────┐   ┌─────────┐   ┌─────────┐  │
│   │ 诊断表   │──▶│ 逻辑链   │──▶│ 悖论自检 │  │
│   │ P0→P3   │   │ 9 链贯通 │   │ 排雷     │  │
│   └─────────┘   └─────────┘   └─────────┘  │
│        │                            │       │
│        ▼                            ▼       │
│   ┌─────────┐              ┌──────────────┐ │
│   │ 改动执行 │──────────────▶│ 收敛判定     │ │
│   │ 述+验   │              │ P0=0 → 通过  │ │
│   └─────────┘              └──────────────┘ │
│                                             │
└─────────────────────────────────────────────┘
```

**铁律**：
- 吃自己的狗粮 — 用 PI 规则审计 PI
- 改一处查全局 — 不留断链
- 加一行问三问 — 重复？冗余？改变 AI 行为？
- 改后必验 — ±5 行回归
- 先读后改 — 禁止基于记忆操作

**校验门禁**：

| 信号 | 判定 |
|------|------|
| P0/P1 = 0 | ✅ 可进入 P2 |
| 连续两轮净改动 ≤2 处 | ✅ 收敛，停止迭代 |
| 行数净增 >10 且无 P0/P1 | ❌ 熵增失控，回顾 |

**改动范围**：

| 文件 | 何时改 |
|------|--------|
| SKILL_META.md | 每轮必审（唯一真源） |
| COMPILER.md | 编译规则需调整时 |
| CHANGE_LOG.md | 每轮结束时 |
| README.md | 影响用户可见概念时（中文主文档，英文版在 README.en.md） |

**保留 vs 剥离 决策树**：

```
这段内容...
    │
    ├── AI 读到后会改变下一个 token 生成？
    │       ├── Yes → ✅ 保留
    │       └── No ──┐
    │                │
    ├── 是认知处理策略（MBTI/认知阵/认知流管线）？
    │       ├── Yes → ✅ 保留（信息不失真）
    │       └── No ──┐
    │                │
    ├── 是格式化输出模板？
    │       ├── Yes → ✅ 保留（用户需看到 PI 生效信号）
    │       └── No ──┐
    │                │
    ├── 是古典引语/思想源/示例/设计rationale？
    │       ├── Yes → ❌ 仅保留在 SKILL_META.md
    │       └── No ──┐
    │                │
    └── 边界注解？
            ├── 影响"何时用/不用"的判断 → ✅ 保留
            └── 纯解释性 → ❌ 剥离
```

**校验门禁**：

| 序 | 检项 |
|----|------|
| 一 | 行为指令无丢失（逐条比对祈使句） |
| 二 | 输出格式模板完整（13+ 种） |
| 三 | 触发条件完整 |
| 四 | description 索引的概念在精简版中均有定义 |
| 五 | emoji 情绪价值适当 |

## 数据流：一次完整迭代

```
用户提出改进方向
        │
        ▼
   ┌─────────┐
   │ P1 迭代  │  iterate.md 协议
   │          │  五维十四检 + 九大逻辑链
   │ 改动 →   │
   │ SKILL_   │
   │ META.md  │
   └────┬─────┘
        │ ✅ P0/P1 = 0
        ▼
   ┌─────────┐
   │ P2 编译  │  COMPILER.md 规则
   │          │  剥离非行为内容，保留认知策略
   │ 产出 →   │
   │ SKILL.md │
   └────┬─────┘
        │ ✅ 行为指令无丢失
        ▼
   ┌─────────┐
   │ P3 分发  │  DISTRIBUTE.md + PURGE.md
   │          │  替换 body → 执行 PURGE → 校验 description
   │ 产出 →   │
   │ 6 平台   │
   │ 文件     │
   └────┬─────┘
        │ ✅ body 一致 + description 一致
        ▼
   ┌─────────┐
   │ P4 翻译  │  TRANSLATE.md 规则
   │          │  行为等价 + 结构一致
   │ 产出 →   │
   │ 英文版   │
   └────┬─────┘
        │ ✅ 行为等价校验
        ▼
   完成 🎯
```

## 触发条件速查

| 场景 | 触发阶段 | 说明 |
|------|---------|------|
| 内容演进（新增能力/优化设计） | P1 → P2 → P3 → P4 | 完整流水线 |
| 编译规则调整 | P2 → P3 → P4 | 跳过 P1，直接重新编译 |
| 仅 description 变更 | P3（仅 frontmatter） → P4 | 只同步 description |
| 仅 body 文案微调 | P3（仅 body） → P4 | 只同步 body |
| PURGE 规则变更 | P3（重新裁剪） → P4 | 重新分发所有适用平台 |
| 新增平台支持 | P3 + P4 | 新增 frontmatter 映射 + 翻译目标 |
| 中文版结构性改动 | P4（必须重新翻译） | 行为等价重校验 |
| 纯文案微调 | P4（可选择性更新） | 按需 |

## 质量守护：贯穿全线的原则

```
┌──────────────────────────────────────────────────┐
│                                                  │
│  🔒 真源唯一    SKILL_META.md 是唯一迭代真源       │
│                 禁止在下游产物中独立修改            │
│                                                  │
│  📐 信息不失真   认知策略（MBTI/认知阵/认知流管线）  │
│                 全链路保留，编译不丢行为信息         │
│                                                  │
│  ⚡ 高信息密度   长而有意义 > 短而无意义             │
│                 不说废话，深度思考后再输出           │
│                                                  │
│  🔄 行为等价    编译/翻译后的产物与真源行为一致       │
│                 祈使句一一对应，触发条件完整         │
│                                                  │
│  ✂️ 按需裁剪    PURGE 在分发阶段执行               │
│                 不修改 SKILL.md 本身               │
│                                                  │
│  🧬 平台适配    各平台 frontmatter 独立            │
│                 body 统一（purged/unpurged）       │
│                                                  │
└──────────────────────────────────────────────────┘
```
