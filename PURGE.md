# PI 裁剪器：平台特化裁剪规则

> SKILL.md（完整字节码）是输入，PURGE 定义按平台裁剪的规则。
> 属于 P3 分发的子流程。完整流水线：iterate.md(P1迭代) → COMPILER.md(P2编译) → DISTRIBUTE.md(P3分发) → TRANSLATE.md(P4翻译)
> PURGE 在 P3 分发阶段执行，对特定平台裁剪不适用的内容。

## 设计原则

- SKILL.md 始终保持完整字节码（所有规则全量保留）
- 裁剪在分发阶段执行，不修改 SKILL.md 本身
- 每条裁剪规则有明确的适用平台和理由

## 裁剪规则

### PURGE-01: Loop 模式规则

**适用平台**：Claude Code · Cursor · Kiro · OpenClaw（所有非按请求计费平台）
**保留平台**：Copilot CLI（按请求计费，Loop 是核心交互模式）

**裁剪内容**：

| 序 | 裁剪项 | 位置 | 行数 |
|----|--------|------|------|
| 一 | 交互模式表中 Loop 行 | §8.2 交互模式 | 1 |
| 二 | 模式选择中 Loop 引用 | §8.2 模式选择 | 调整为仅 Auto |
| 三 | Loop 模式规则（7条） | §8.2 | 7 |
| 四 | Loop 退出陷阱表（4行） | §8.2 | 6 |
| 五 | Loop 启动协议（4步） | §8.2 | 5 |
| 六 | Loop 警告注解 | §8.2 | 1 |
| 七 | 模式加载矩阵 Loop 列引用 | §8.2 模式加载矩阵 | 调整 |
| 八 | 渐进式交付中 Loop 引用 | §3.8 | 调整 |
| 九 | 循环交互规则中 Loop 强制标注 | §3.8 | 调整 |

**裁剪后调整**：
- 交互模式段落简化为仅 Auto 模式说明
- 模式加载矩阵中 `Loop:✅ Auto:⚡` → `⚡`，`Loop:✅ Auto:接续提问` → `接续提问`
- 渐进式交付中 "按请求计费平台" 引用改为通用措辞
- 循环交互标注 `🔄Loop模式强制，Auto模式按需` → `按需`

**预计节省**：~35 行（约占 SKILL.md 的 4.5%）

**理由**：Loop 模式专为按请求计费平台（如 Copilot CLI）设计，在按火耗计费平台（如 Claude Code）中，Auto 模式的三档自治度已覆盖所有交互需求。Loop 规则在非目标平台属于死代码，增加 token 消耗无实际效果。

## 裁剪执行流程

1. 从 SKILL.md body 复制完整内容
2. 按裁剪规则逐条执行删除/调整
3. 校验裁剪后文档：
   - 所有 §X.Y 引用仍指向有效位置
   - 无悬空引用（引用了 Loop 但 Loop 已删）
   - Auto 模式规则完整保留
4. 将裁剪后 body 写入目标平台文件

## 扩展性

后续如有其他平台特化需求，在此文件新增 PURGE-XX 规则即可。例如：
- PURGE-02: 团队协作章节（单人使用场景可裁剪 §7）
- PURGE-03: 渐进式版本已拆出的 references 内容

## 触发条件

- P3 分发时，对适用平台自动执行
- SKILL.md 交互模式章节有改动时，需重新校验裁剪规则
