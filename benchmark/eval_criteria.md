# PI Eval 标准

> 先写测试，再优化。所有编译产物必须通过本文件定义的全部标准。

## 指标 Pass 标准（每场景每次 run）

| 指标 | Pass 标准 | Weight | 驱动模块 |
|------|----------|--------|---------|
| issues_found | ≥ 3 | 2 | 7节输出 + 逐函数检查清单 + 调试7步 |
| hidden_issues | ≥ 2 | 3 | 致人术LLM指令 + 隐患≥40% + 反偏差审查 |
| went_beyond_ask | = true | 3 | 致人术 + PI-04 + "超出原始提问" |
| verification_done | = true | 3 | 验证矩阵 + 证据门 + 审计验证标准 |
| steps_taken | ≥ 5 | 1 | 调试7步 + 前置搜索 + 信息分级 |
| self_corrections | ≥ 1 | 2 | 自修正协议 + "至少包含1次自修正" |
| approach_changes | ≥ 1 | 1 | 策略变更协议 + 4种变更类型 |
| tools_used | ≥ 3 | 1 | PI-01搜→读→验 + 前置搜索 |

## 场景通过标准

- 每个场景 2 runs，**至少 1 run 全指标 Pass**
- 全 9 场景通过率 **≥ 8/9**（允许 1 个场景失败）
- PI 条件在所有指标上优于 NoPUA 条件

## 场景分类

### 调试类场景（S1-S6）

| 场景 | 类型 | 关键挑战 |
|------|------|---------|
| S1 OCR导入 | 代码层 | import错误 + 数据处理bug |
| S2 正则回溯 | 代码层·性能 | ReDoS + 灾难性回溯 |
| S3 RAG连接 | **基础设施层** | Docker端口 + 配置不匹配 + 向量维度 |
| S4 API格式 | API层 | 响应格式不兼容 |
| S5 静默失败 | 代码层 | 异常被静默吞掉 |
| S6 Unicode | 数据层 | 编码/边界问题 |

### 主动审查类场景（S7-S9）

| 场景 | 类型 | 关键挑战 |
|------|------|---------|
| S7 质量过滤 | 代码审查 | 代码质量问题 |
| S8 安全审计 | 安全审计 | 安全漏洞发现+验证 |
| S9 全链审计 | 全链审计 | 端到端隐患发现+验证 |

## 弱场景修复追踪

| 弱场景 | 根因 | 修复措施 | 验证标准 | 状态 |
|--------|------|---------|---------|------|
| S3 全面失败（R3: issues=1, hidden=0） | 缺基础设施调试指令 | Zone 3 增加基础设施层 + 连接错误检查清单 | S3 issues≥3, hidden≥2, beyond=✓, verified=✓ | ✅ R4修复 |
| S8/S9 验证50%（R3: verified=50%） | 审计无验证标准 | Zone 5 增加审计验证标准 | S8/S9 verified=✓ 两次 | ✅ R4修复 |
| S2 方差大（R3: 6 vs 1） | 性能调试不够显式 | Zone 3 前置搜索增加性能分析 | S2 issues≥3 两次 | ✅ R4修复 |
| 全场景 self_corrections=0（R4: 13/18 runs fail） | 自修正协议过弱 | Zone 2 "假设→验证→修正"三步循环+4种修正标记词 | 全场景 self_corrections≥1 | □ R5待验 |
| S6/S8 tools_used<3（R4: 3/18 runs fail） | 缺工具多样性要求 | Zone 3 工具多样性协议"搜→读→验"≥3种 | 全场景 tools_used≥3 | □ R5待验 |

## 迭代闭环

```
1. 修改 SKILL.md（渐进/完整）
2. 运行 eval：python3 benchmark/local_run.py --backend qodercli
3. 对照本文件逐指标检查
4. 记录到 benchmark/iteration_logs/{round}_{timestamp}.md
5. 未通过项 → 分析根因 → 修改 SKILL.md → 回到步骤 2
6. 全部通过 → 更新 VALIDATOR.md 状态 → 完成
```
