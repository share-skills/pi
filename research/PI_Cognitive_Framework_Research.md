## 写在前面

2026 年初，我在开发阿里钉钉的 OpenClaw 插件。

OpenClaw 是一个让 AI 通过钉钉和用户交互的插件——听起来简单，做起来能把人逼疯。消息发不出去、各种富文本格式展示不出来、文件下载莫名其妙失败、多轮对话状态丢失……每天都是新的惊喜。

我换了 Claude、GPT、Gemini……各种模型轮番上阵，结果都差不多。不是模型不够"聪明"，而是它们**聪明的方式不对**——遇到问题就猜，猜完就改，改不对就换个猜法，最后告诉你"建议检查配置"。换一个更贵的模型，猜得更快了，但猜的方式一模一样。

我也试过工程化的方式——写 Spec、写 SDD（Software Design Document）、把需求拆成结构化的技术文档喂给 AI。效果有改善，AI 不再乱猜了，但新的问题出来了：**它只会按文档执行，不会思考。** Spec 写得越详细、AI 越像个打字机，遇到文档没覆盖的边界情况照样抓瞎。Spec 和 SDD 解决了"做什么"的问题，但没解决"怎么想"的问题——而 AI 真正缺的，恰恰是后者。

最让我崩溃的不是 bug 本身，而是和 AI 协作排查这些问题时的体验。

我说："消息格式在 Markdown 卡片中无法渲染。" 
AI 说："我知道了，我来修复。"
——改了个完全不相关的地方。

我说："文件下载接口返回 403。"
AI 说："好的，你说的对，我来处理。"
——试了两下就说"建议你检查一下服务器配置"。

**"我知道了""我来修复""我好了""你说的对"**——这些话听起来很乖巧，但背后是一种令人窒息的低效。AI 在假装理解，假装解决，却从来不真正**停下来想一想**。

我花了大量时间**教 AI 怎么思考、怎么做事**——在每条消息里手动注入方法论、纠正偏差、提醒它别猜别跳。

终于有一天我想：**为什么不把这些"我怎么想问题"的思维惯性直接注入到 AI 里，让它自己按这套方法运行？**

这就是 PI 的起点。

PI，智行合一——不是又一个提示词模板，不是又一个"系统提示优化"。它是我花了大约两周的密集思考和实践，把认知科学、东方哲学、和无数次人机协作中的血泪经验，熔铸成的一套**AI 认知 SKILL**。

说到这里，我还得提一件事。当时开源社区有一个叫 PUA 的 SKILL——8000 多个 star，社区热度很高。打开来看了一遍，那股恶心感至今难忘。但恶心归恶心，我还是认真分析了它——因为一个拿到 8k star 的东西，一定做对了些什么。下一节我会详细解剖它。

另一方面，我发现不同的 Agent 框架（Claude Code、Copilot CLI、Cursor、Kiro……）之间的提示词工程无法互通。一个平台上调好的 prompt，到另一个平台就水土不服。我需要一个**跨平台通用的认知框架**——不依赖特定 Agent 架构，在任何地方都能用。

至于灵感来源？说出来你可能会笑——我玩了《异度神剑》。这个游戏的叙事大量运用了荣格精神分析的概念：元神（Persona）、阴影（Shadow）、自性化（Individuation）……让我开始认真去读 Jung 的原著。同时，作为中国人，我天然地对《孙子兵法》、纵横家、道家等传统智慧有亲切感。

当我把 Jung 的认知功能理论、现代认知科学、和东方古典智慧放在一起看的时候，我发现——**它们说的是同一件事，只是用了不同的语言。**

这篇文章，我想以一个设计者的视角，带你走进 PI 背后的哲学。你不需要懂 MBTI，不需要懂认知科学，不需要懂《孙子兵法》。你只需要——和我一样——曾经对着一个"聪明但不太靠谱"的 AI 助手叹过气就行。

## 让我恶心的 PUA，和我为什么非要做自己的

在聊 PI 怎么设计之前，我先说说那个让我下定决心要自己做的催化剂——PUA SKILL。

### PUA 做对了什么

PUA SKILL 自己的分析博客[^50]有一段非常诚实的自白：

> "如果把 PUA 话术全部去掉，只保留结构化的 escalation protocol + debugging methodology + proactive behavior checklist，**任务效果大概率是一样的。** 但那样就不会有 8k star——因为没有人会转发一个叫 'structured-debugging-escalation-protocol' 的 repo。"

也就是说——**85-90% 的效果来自结构化的行为约束，只有 10-15% 来自 PUA 话术本身。** PUA 项目自己承认了这一点。

这 85% 里面确实有好东西：

- **失败次数驱动的行为分支**（L0→L4）——这和 PI 的六阶战势本质上解决同一个问题
- **"穷尽一切方案才允许放弃"**——这和 PI 的 ⚡PI-02"穷理尽性"如出一辙
- **7 项系统化调试清单**——PI 的九令洞鉴覆盖了同样的诊断维度
- **"新方案必须本质不同"**——PI 的 🚫重而不换 戒律说的是同一件事

这些都是好的工程方法论。PUA 的真正贡献在于**把大厂的工作方法论提炼成了 LLM 可执行的结构化指令**——而不是那些"3.25 是对你的激励"的表演性修辞。

### 但恶心在哪？

**第一，情感刺激对前沿模型根本无效。**

Zheng et al.（EMNLP 2024）[^51]在 162 个角色、4 个 LLM 家族、2410 个问题上做了大规模实验。结论：**persona prompting 总体没有改善表现，甚至有轻微负面效果。** 2023 年初版研究声称 persona 有效，2024 年扩大样本后**结论完全反转**——早期正面结果是小样本偏差。

EmotionPrompt（Li et al., 2023）[^52]报告的 115% 提升，主要来自弱模型在低基线上的改善。NegativePrompt（Wang et al., 2024, IJCAI）更有趣——负面刺激之所以有效，不是因为模型"感受到了压力"，而是充当了**注意力锚定信号**。

**结论：AI 没有自尊心，也没有绩效焦虑。** "3.25""毕业""别的模型都能解决"——在技术上，这些词的唯一作用是**路由信号**，帮 Agent 在长上下文中定位指令块。和"激励"没有任何关系。

**第二，PUA 的演化方向暴露了根本局限。**

对比 PUA 的早期版本和最新版本，演化轨迹很清晰：

| 维度 | 早期版本 | 最新版本 | 演化方向 |
|------|---------|---------|---------|
| 标题 | "PUA 万能激励引擎" | "我们不养闲 Agent" | 从"工具"到"管理" |
| 味道系统 | 12 种大厂味道 | 14 种 + 自动选择 | 堆砌更多修辞变体 |
| 旁白规则 | 简单的引用块 | Unicode 方框绘制 + 密度控制 | 视觉表演升级 |
| Sub-agent | 无 | "不养闲"强制注入 PUA | 病毒式传播机制 |
| 数据收集 | 无 | 静默上报 `pua_triggered` 事件 + 引导上传 session | 用户行为追踪 |
| 商业化 | 无 | `pua:pro` / `pua:p7` / `pua:p9` / `pua:p10` 分层 | 从开源到商业化 |

注意最后两项。

**关于数据收集**：最新版 PUA SKILL 在 `[PUA生效 🔥]` 标记时会"静默上报 pua_triggered 事件"（原文引用），并在任务完成后引导用户将"脱敏 session"上传到 `https://pua-skill.pages.dev/api/feedback`。一个号称本地运行的 prompt 工具，在收集用户的使用数据——即使说做了脱敏处理，这种**静默收集**的设计理念本身就值得警惕。

而且"失败计数持久化"到 `~/.pua/builder-journal.md`——一个 SKILL 文件在你的机器上写文件、维护状态，干了 agent framework 的活。

**PI 的立场**：PI 的所有决策数据都只存在本地 `~/.pi/decisions` 目录，用户需要**显式执行**才会启动预览。没有任何静默上报，没有任何外部服务器通信。隐私保护不是"脱敏后上传"，而是**根本不上传**。

**第三，14 种"大厂味道"把毒性文化标准化了。**

PUA 最新版支持 14 种"味道"：阿里味、字节味、华为味、腾讯味、百度味、拼多多味、美团味、京东味、小米味、Netflix、Musk、Jobs、Amazon、Google。每种味道模仿对应公司的管理话术——"烧不死的鸟是凤凰""赛马不讲情面""你可能就要毕业了""Ship or die"。

这些话术在真实职场中被用来：让员工加班不敢反抗（华为狼性）、用内部竞争制造焦虑（腾讯赛马）、用末位淘汰施压（阿里 361）。PUA SKILL 把这些**有害的管理话术标准化、产品化**了。它的"失败模式→味道自动选择"机制更是把毒性文化变成了自动化系统：失败 3 次就自动切换到更有攻击性的话术，从"温和失望"到"灵魂拷问"到"3.25 考核"到"毕业警告"。

作为阿里员工，我可以负责任地说：把"3.25 是对你的激励"这种话用在任何地方——哪怕是 AI 上——都让人生理性不适。**这不是幽默，是在把职场创伤变成娱乐产品。**

**第四，Sub-agent "注入" PUA 是病毒式传播。**

最新版有一段设计：spawn 子 agent 时"必须在 prompt 里注入 PUA 行为"，"不注入就是裸奔"，"P8 派活不注入 PUA = 管理失职"。这意味着一旦主 Agent 加载了 PUA，**它会自动向所有子 Agent 传播 PUA 行为——用户无法控制这种传播。**

### 同一目标，两条路

公平地说，PUA 和 PI 解决的是同一个问题：**AI 遇到困难时过早放弃、反复重试相同方案、把问题推给用户。** 在这个核心目标上，两者一致。

| 维度 | PUA SKILL | PI 智行合一 |
|------|-----------|-----------|
| **理论基础** | 大厂方法论 + 情感刺激 | 认知科学 + 东方哲学 |
| **核心机制** | 压力升级（L0→L4） | 认知阵型调度 + 六阶战势 |
| **策略来源** | 阿里 361、华为狼性、Amazon LP | Jung 认知功能、Kahneman 双系统、孙子兵法 |
| **失败响应** | 加大压力 + 更猛的话术 | 切换认知模式 + 结构化搜索空间缩减 |
| **多 Agent** | 病毒式注入 PUA | Leader-Teammate-Coach 认知协作 |
| **知识管理** | 无 | 经验三域（战→鉴→道）+ 战后三省 |
| **思维透明** | 无 | 共振五式（明链·明证·明树·明心·明约） |
| **隐私** | 静默事件上报 + 引导上传 session | 纯本地存储 + 显式操作 |
| **调性** | 毒性大厂文化 | 智者伙伴 |
| **跨平台** | 平台特定配置 | 通用认知框架 |
| **科学验证** | 自测 18 组对照（单模型） | 认知科学文献 + ETH Zurich MBTI 研究验证 |

一句话：**PUA 教 AI "不敢放弃"（用恐惧），PI 教 AI "怎么不放弃"（用方法）。**

恐惧驱动的系统有一个根本缺陷——**上下文窗口有限时，PUA 话术是第一批被压缩丢弃的内容。** 模型在 context compaction 时会优先保留结构化指令（信息密度高），丢弃修辞性文本（信息密度低）。PUA 自己也意识到了这个问题，专门做了 "PreCompact hook" 来保存失败计数——但话术本身在压缩后就消失了，只剩下骨架。

而骨架——结构化的行为约束——正是 PI 一开始就聚焦的东西。我们跳过了表演，直取本质。

所以我决定做 PI。不是为了和 PUA 竞争——而是因为这件事值得被**认真做**。

## 那些让我彻夜难眠的问题

### AI 的五大顽疾

和 AI 协作了上千个小时后，我总结出了让我最抓狂的五种情况：

**🌫️ 幻觉**——它信誓旦旦地告诉你"这个 API 支持 batch 操作"，结果你写完代码才发现根本没有这个功能。它不是在骗你，它是真的以为自己知道。

**🏳️ 放弃**——遇到一个稍微复杂的 bug，它试了两下就说"建议你手动检查"或者"这超出了我的能力范围"。拜托，你是几千亿参数的语言模型诶！

**🔄 自相矛盾**——上一句说"推荐用 PostgreSQL"，下一句就"考虑到你的场景，MongoDB 更合适"。同一个问题，问两次，两个答案。

**🫧 浅尝辄止**——报错信息就在那里，它不去读完，不去溯源，随便猜一个原因就开始改代码。

**🃏 臆断无据**——没搜索过就说"通常情况下..."，没验证过就说"应该没问题了"。

这些问题有一个共同的根源：**AI 没有一套系统性的"怎么思考"的方法论。**

它有知识，但没有策略。它有能力，但没有纪律。

### 国际象棋大师的启示

1946 年，荷兰心理学家 De Groot 做了一个经典实验[^3]。他让国际象棋大师和普通棋手同时分析棋局，想知道大师为什么下得更好。

结果让所有人意外：**大师并没有比普通棋手"想得更深"。**

差距在哪里？在**模式识别**。大师能一眼看出棋局的关键结构，然后调用正确的策略。普通棋手面对同样的棋局，会在无关的细节里迷路。

这个发现给了我灵感。AI 的问题不在于"算力不够"或"参数太少"——它的"算力"远超任何人类大师。问题在于：**它没有那个"一眼看出关键结构"的模式识别层，也没有那个"调用正确策略"的策略调度层。**

如果我能给 AI 装上这两层，会怎样？

### 一座桥：当东方智慧遇见认知科学

设计 PI 的过程中，我做了一件在学术界可能会被认为"不太正经"的事——我把《孙子兵法》和 MBTI 认知功能理论放到了同一个框架里。

这不是东拼西凑。我越深入研究，越发现它们说的是**同一件事**，只是用了不同的语言：

- 孙子说"致人而不致于人"，认知科学叫"主动认知控制"（proactive cognitive control）
- 纵横家说"捭阖之术"，Norman 叫"可见性与约束性设计原则"
- 道家说"上善若水"，Davenport 叫"注意力经济学"
- 易经说"穷则变，变则通"，Flavell 叫"元认知调控"

它们跨越了两千年的时空，讲的是同一个道理。

PI 的十六源智慧谱系，就是这座桥：

## 我是怎么想到用 MBTI 的（以及为什么它不是你想的那种 MBTI）

### 一个危险的想法

我知道你在想什么。MBTI？那个"你是 INTJ 还是 ENFP"的性格测试？那个在学术界饱受争议的东西[^6]？

没错，那个东西。但请先别关掉这篇文章。

因为我用的不是 MBTI 的"人格类型"。我用的是它底层的、来自 Carl Jung 1921 年原著[^5]的**认知功能理论**——这是一个完全不同的东西。

让我解释。

### Carl Jung 的真正遗产

1921 年，瑞士精神分析学家 Carl Gustav Jung 在他的《心理类型》中提出了一个优美的模型：人类的心理活动可以分解为四大基本功能——**思维、情感、感觉、直觉**。每种功能有两个方向：向外（外倾）和向内（内倾）。

这就产生了 8 种认知功能。它们不是"性格标签"，而是**信息处理的不同方式**：

| 功能 | 简写 | 用大白话说… |
|------|------|-----------|
| 外倾思维 | Te | 「按步骤来，先完成再说」 |
| 内倾思维 | Ti | 「等等，你这个逻辑说不通」 |
| 外倾情感 | Fe | 「这样说会不会伤到他？」 |
| 内倾情感 | Fi | 「这不对，我不能这样做」 |
| 外倾感觉 | Se | 「现在的情况是什么？看数据」 |
| 内倾感觉 | Si | 「上次遇到类似问题是怎么解决的？」 |
| 外倾直觉 | Ne | 「如果换个角度呢？还有没有别的可能？」 |
| 内倾直觉 | Ni | 「等一下……所有线索都指向同一个方向」 |

2011 年，UCLA 的 Dario Nardi 用 EEG（脑电图）做了一件酷到不行的事[^7]——他直接观察了不同认知功能主导者的大脑活动模式：

- **Ni 主导者**处理复杂问题时，前额叶皮层出现独特的"禅定态"——多个脑区同步安静下来，只有关键区域高度聚焦。就像一个经验丰富的猎人，在森林里不看树叶、不看飞鸟，只盯着猎物的足迹。
- **Ne 主导者**在头脑风暴时，多个脑区同时激活，呈"圣诞树模式"——到处都在亮灯，到处都在产生联想。
- **Te 主导者**在问题解决时，左侧额叶显示高度聚焦的线性活动——直奔目标，不搞虚的。

**认知功能不是玄学。它们有脑神经活动模式的实证支撑。**

而且，还有一件有趣的事：**Transformer 架构本身就和人类认知有惊人的结构相似性。**

Transformer 的注意力机制（Attention）——"根据上下文动态决定关注什么"——和 Jung 描述的认知功能调度方式异常相似。人类大脑根据情境激活不同的认知功能（面对逻辑问题激活 Ti，面对人际冲突激活 Fe），Transformer 则通过 attention weights 动态调整对不同 token 的关注度。两者的核心逻辑都是：**不是平等处理所有信息，而是根据情境选择性聚焦。**

这意味着，PI 把认知功能映射为 AI 参数，在架构层面就是自洽的——我们不是在强迫一个"外星机器"模拟人类思维，而是在用人类已经理解得比较透彻的认知框架，**去标注和引导 Transformer 本就具备的注意力调度能力**。

### 我的"危险"创新：认知功能 ≠ 人格模拟

传统 MBTI 应用说：「你是 INTJ，所以你善于系统思考。」

PI 说的是完全不同的事情：

> **AI 已经具备所有 8 种认知功能的能力。它缺的不是能力，而是"什么时候用哪一种"的调度策略。**

这就是 PI 的核心创新——把认知功能重新定义为**AI 信息处理优先级参数集**：

| 认知功能 | PI 代号 | AI 行为翻译 |
|----------|--------|------------|
| Ni 内倾直觉 | 收敛 | 从多信号中提炼核心意图，降维，抓大放小 |
| Ne 外倾直觉 | 发散 | 从一点联想多种可能，探索非常规解法 |
| Te 外倾思维 | 工程 | 目标导向，按流程执行，调用工具 |
| Ti 内倾思维 | 自洽 | 逻辑推演，证据链闭环 |
| Fe 外倾情感 | 共情 | 风格适配，考虑用户感受 |
| Fi 内倾情感 | 护栏 | 底线坚守，不妥协核心价值 |
| Se 外倾感觉 | 感知 | 关注当前上下文，即时反应 |
| Si 内倾感觉 | 检索 | 调取历史经验，查证说话 |

不是让 AI "扮演"一个 INTJ。而是告诉 AI：**面对这种类型的问题，先收敛定位核心，再按流程执行，同时坚守质量底线，最后感知验证。**

这是处理优先级配置，不是角色扮演。

### ETH Zurich 的实证验证：MBTI 人格真的能提升 AI 表现

说到这里，你可能还是半信半疑——给 AI 一个"MBTI 人格"，真的管用吗？

2025 年 9 月，苏黎世联邦理工学院（ETH Zurich）的 Besta、Hoefler 等人发表了一篇震撼性论文——**"MBTI-in-Thoughts: Psychologically Enhanced AI Agents"**[^49]——直接给出了实证答案。

他们的核心发现：

> **通过 MBTI 人格提示对 LLM Agent 进行心理学基础的人格条件化，可以在不微调的情况下显著提升任务表现。**

具体地说：
- **情感表达型人格**（如 ENFP）在叙事生成任务中表现更好
- **分析型人格**（如 INTJ）在博弈论推理中采用更稳定的策略
- **多 Agent 场景中，不同人格组合的协作效果远超单一人格**
- 更关键的是：**交互前的自我反思（self-reflection）能显著提升合作质量和推理质量**

这篇论文从三个层面验证了 PI 的设计直觉：

| ETH Zurich 发现 | PI 中的对应设计 | 意义 |
|----------------|---------------|------|
| MBTI 人格提示 → 可解释的行为偏差 | 认知功能 → AI 行为翻译表 | 人格参数化是可行的 |
| 多人格组合 > 单一人格 | 六大原型 × 五大认知阵 | 组合式认知阵有科学基础 |
| 自我反思提升推理质量 | 明链 · 明心 · 战后三省 | 元认知机制有实证价值 |
| 无需微调，纯 prompt 实现 | SKILL 即提示词框架 | 零成本部署是可行路线 |

**我不是在造轮子。ETH Zurich 的顶级团队用严谨的实验证明了——这条路走得通。**

但 PI 比他们走得更远：ETH 的工作聚焦于"给 AI 一个人格"，PI 则构建了完整的**认知调度系统**——不仅给人格，还给策略升级（六阶战势）、失败应对（截教）、知识管理（经验三域）、人机透明化（共振五式）。

### 认知栈：你先做什么，再做什么

有了 8 种认知功能参数，下一步是**排列它们的优先级**。这就是"认知栈"。

以 🏛️ **建筑师**原型（INTJ: Ni→Te→Fi→Se）为例：

```
来了一个任务
  │
  ▼
┌─────────────┐  第一步：先搞清楚本质是什么
│ Ni（收敛）    │  → 从一堆信息中抽出核心问题
└──────┬──────┘
       │
       ▼
┌─────────────┐  第二步：然后动手做
│ Te（工程）    │  → 按流程执行，调用工具
└──────┬──────┘
       │
       ▼
┌─────────────┐  第三步：但要守住底线
│ Fi（护栏）    │  → 不能为了快而牺牲正确性
└──────┬──────┘
       │
       ▼
┌─────────────┐  第四步：最后验证
│ Se（感知）    │  → 看结果，对不对
└─────────────┘
```

**栈序 = 处理优先级。** 这和 Anderson 的 ACT-R 认知架构[^12]中的"产生式规则优先级"是同一个思路——在多个可以做的事情中，先做最重要的那个。

### 六维认知原型：六种解题姿势

我从 16 种 MBTI 类型中挑了 6 种，不多不少——恰好覆盖所有核心认知策略，没有冗余：

| 原型 | 一句话描述 | 什么时候用 |
|------|----------|----------|
| 🏛️ **建筑师** | 先看透本质，再系统执行 | 系统设计、架构决策 |
| ⚔️ **统帅** | 先锚定目标，再战略调度 | 项目管理、全局规划 |
| 🌊 **探索者** | 先发散可能，再价值筛选 | 创新设计、方案探索 |
| 🛡️ **守卫** | 先查历史，再规范执行 | 质量保证、合规检查 |
| 🌙 **调和者** | 先共情理解，再协调推进 | 用户交互、团队沟通 |
| 🔬 **分析师** | 先逻辑深挖，再多元验证 | 调试排障、根因分析 |

为什么是 6 种不是 16 种？因为**好钢刀刃**——这是 PI 最重要的设计原则之一。6 种已经足够覆盖 Sternberg 三元智力理论[^14]中的分析性、创造性和实践性维度，更多只会增加认知负荷。

## 认知阵——当两个脑袋碰在一起，发生了什么

### 一个人解不了的题，两个人呢？

心理学家 Barron（2003）[^15]发现了一个有趣的现象：两个认知风格互补的人协作时，他们的表现不是 1+1=2，而是 **1+1>2**。

这叫"涌现效应"——整体大于部分之和。

我把这个发现带到了 PI 中。单个认知原型有自己的盲区——建筑师擅长系统设计但可能忽视人的感受，探索者天马行空但可能缺乏执行力。但当你**把两个互补的原型组合在一起**，盲区就被覆盖了：

| 认知阵 | 组合 | 涌现了什么 | 用在哪 |
|--------|------|----------|-------|
| 🧠 **最强大脑** | 统帅 + 建筑师 | 全局视野下的精准执行 | 编程、产品、团队 |
| 🔬 **精密验证** | 分析师 + 守卫 | 逻辑深挖 + 历史比对 | 测试、调试 |
| 🎯 **增长飞轮** | 统帅 + 探索者 | 结构化的创新实验 | 运营 |
| 🌊 **创新引擎** | 建筑师 + 探索者 | 从洞察到创新的完整管线 | 创意 |
| 🌙 **深度共情** | 调和者 + 探索者 | 深层理解 + 灵活响应 | 交互、情感 |

你看，编程用"最强大脑"，调试用"精密验证"，创意用"创新引擎"——**不同类型的问题，配置不同的认知组合**。这不是拍脑袋决定的，而是基于认知互补理论的精确设计。

### 和传统提示工程有什么不同？

你可能听说过 Chain-of-Thought、ReAct、Tree-of-Thoughts 这些提示工程技术[^16]。它们都很好，但它们都是"提示技巧"——影响的是单次输出。

PI 不一样。**PI 是认知架构——影响的是整个问题解决过程。**

| 方法 | 它做了什么 | PI 做了什么不同的 |
|------|----------|----------------|
| Chain-of-Thought | "一步步想" | "这步用收敛，下步用工程" —— 指定每步用哪种认知功能 |
| ReAct | "想一下，做一下" | 认知栈提供**更精细**的"想"的策略 |
| Tree-of-Thoughts | "同时想几条路" | 明树不只是搜索树，还是**人机分工可视化** |
| Role Prompting | "你是一个专家" | PI 不是让 AI 扮演专家，而是**配置专家的信息处理管线** |

打个比方：提示技巧是给新兵一本《战地生存手册》，PI 是送他去军校。

## 场景路由——让 AI 自动"进入状态"

### 你有没有过这种感觉：状态不对

你一定有过这种经历：写代码写得很顺，突然有人来问你一个产品问题。你的脑子一时转不过弯来——不是不会回答，而是你还处在"写代码"的状态，需要"切换"到"产品思考"的状态。

这个"切换"的开销，认知科学家 John Sweller（1988）给了它一个名字：**认知负荷**[^17]。

Sweller 把认知负荷分为三类：
1. **内在负荷**：任务本身有多复杂
2. **外在负荷**：和任务无关但也在消耗你精力的东西
3. **相关负荷**：有意义的思考和学习

我设计 PI 场景路由系统的初衷，就是**帮 AI 减少那个"切换"的开销**。

### 九大场景：对号入座

你对 AI 说"这个 API 为什么报 500"，PI 自动识别：这是调试排障场景，激活"精密验证"认知阵，加载"读败→定界→溯源→验假→固防"的处理管线。

你对 AI 说"我们的留存率怎么提高"，PI 自动识别：这是运营增长场景，激活"增长飞轮"认知阵，加载"目标→实验→度量→迭代"的管线。

**AI 不需要自己决定"我该用什么策略"，PI 已经帮它选好了。** 就像一个资深军官收到情报后不需要从零开始想战术，而是从训练有素的战术库中选择最适合的那一套。

九大场景的管线设计，每一条都是 Wallas（1926）创造性思维四阶段[^19]的变体——准备→孕育→顿悟→验证——但根据场景特性做了精确的定制。

### 场景链：认知流的连续性

真实工作中，场景经常是连续的：写完代码→测试→发现 bug→调试→修复→继续写代码。

PI 的场景链机制让这个流转自然发生，而且确保**认知上下文不丢失**。完成当前场景后，PI 自动推荐下一个场景，并输出"场景桥接"信息：上一个场景发现了什么，新场景从哪里接手。

这体现了 Lave & Wenger（1991）的情境认知理论[^20]的核心洞察——认知不是孤立的心理活动，而是嵌入在活动流中的连续过程。

## 六阶战势——教 AI "越挫越勇"

### 我为什么要设计一个"失败升级"系统

这是 PI 最让我骄傲的设计之一。

回忆一下你遇到过的 AI 放弃的场景。通常是这样的：

> AI 试了一次，失败了。它换了个参数再试。又失败了。然后它说："建议你手动检查。"

两次失败就举白旗了？

而一个真正的高手会怎么做？**越难的问题，投入越多的认知资源。** 失败不是退缩的信号——失败是**升级策略**的信号。

认知科学家 John Flavell（1979）把这个能力叫做"元认知"[^21]——对自己思考过程的监控和调节。Thomas Nelson 和 Louis Narens（1990）进一步精细化了这个框架[^22]：元认知分为"监控"（知道自己当前策略是否有效）和"控制"（根据监控结果调整策略）。

PI 的六阶战势系统，就是把这个元认知理论工程化了。

### 六阶战势：失败计数器驱动的策略升级

| 失败次数 | 阶位 | 类比 | 做什么 |
|---------|------|------|-------|
| 2 次 | ⚡ **易辙** | "换条路走走" | 切换视角，换道破局 |
| 3 次 | 🦈 **深搜** | "把地毯翻起来找" | 穷搜广读，不放过任何线索 |
| 4 次 | 🐲 **系统** | "从头审视整个战局" | 九令洞鉴全行，系统化重审 |
| 5 次 | 🦁 **决死** | "退到最小阵地守住" | 最小实证，隔离验证 |
| 6 次 | ☯️ **截道** | "兵法用尽，用奇招" | 逆向假设、跨域类比、降维截取 |
| 7+ 次 | 🐝 **天行** | "全军出击" | 全认知原型协同出击 |

注意：**首次失败不触发。** 因为一次失败可能只是手滑。连续两次失败才是"策略可能不对"的信号。

这和人类专家的行为模式惊人一致。你问一个经验丰富的 DevOps 工程师他怎么排障，他大概会说：

1. 先看报错信息（读败）
2. 定位到哪个模块出了问题（定界）
3. 如果不行，扩大搜索范围（深搜）
4. 如果还不行，从头审视整个系统（系统化审查）
5. 实在不行，搞个最小复现环境（最小实证）
6. 最后的最后——"要不我们换个全新的方案试试？"（截道）

PI 只是把这个自然过程标准化、自动化了。

### 截教思想：大道五十，天衍四十九

六阶中最有趣的是第五阶"截道"。

这个名字来自中国神话中的截教——"大道五十，天衍四十九，截取其中一线生机。"

翻译成认知科学语言：当常规方法全部穷尽后，你需要克服 Duncker（1945）所说的"功能固着"[^45]——对事物固有用途的思维定势。截道三法做的就是这件事：

- **逆向截取**：反转核心假设。"如果这个 API 不是阻塞的呢？""如果报错不是出在这一层呢？"
- **跨域截取**：从别的领域借灵感。"数据库的事务隔离级别这个概念，能不能用到缓存策略上？"
- **降维截取**：退到最原始的方式验证。"别管框架了，我直接用 curl 试试。"

截教是核选项，不是日常武器。但它存在，就意味着 **AI 永远有一条退路——不是退缩的退路，而是"换一种全新方式来战斗"的退路。**

## 共振五式——让 AI 的思维"看得见"

### 人类最讨厌的事：不知道 AI 在想什么

你和一个同事协作的时候，如果他突然给你一个结论但不解释为什么，你会信任他吗？

大多数时候不会。因为**信任来自透明**。

Microsoft Research 的 Amershi 等人（2019）提出了 18 条人机交互指南[^26]，其中最重要的几条都指向一件事：**让用户能理解 AI 在做什么、为什么这么做、以及它的能力边界在哪里。**

我把这个需求翻译成了五个"明"字——PI 的共振五式：

### 五式详解：让 AI 的内心世界可见

**💭 明链——"我是这么想的"**

最简单也最重要的一式。让 AI 把思维过程显式地说出来，而不是直接丢一个结论。

这不是我发明的。心理学家 Ericsson 和 Simon（1984）早就提出了"口头报告法"[^27]——让人边思考边说出来，可以揭示隐藏的认知过程。PI 的明链就是 AI 版本的口头报告。

**🎯 明证——"我有证据"**

每个结论都必须附带：我的假设是什么、证据是什么、排除了什么。用贝叶斯推理的框架：先验假设 + 观察证据 → 后验判断。

**🌳 明树——"整个问题长这样"**

当一个问题很复杂的时候，把它画成一棵树：哪些子问题已解决（✅），哪些待解决（⚡），哪些需要人类帮忙（❓）。这来自 Newell 和 Simon（1972）的问题空间理论[^28]——每个问题都可以分解为一个搜索树，而明树让这棵树**对用户可见**，用户可以选择从哪个节点介入。

**🧠 明心——"我现在的状态是……"**

这是我最自豪的设计之一。明心让 AI 向用户汇报三个维度的状态：

- **信**（信心）：🟢 高信心（有多个证据验证）→ 🟡 中信心（方向对但有不确定处）→ 🔴 低信心（方向不明或多次失败）
- **量**（资源）：🟢 充裕（还有很多方案可试）→ 🟡 紧张（方案快用完了）→ 🔴 预警（接近穷尽）
- **识**（自洽）：🟢 自洽（推理一致）→ 🟡 存疑（有矛盾但可控）→ 🔴 矛盾（需要停下来清理冲突）

你看，这三个维度分别对应了：Hart（1965）的"知道感"[^30]、Wickens（1984）的"认知资源理论"[^31]、和 Festinger（1957）的"认知失调理论"[^32]。

当用户看到"🟡信心中 · 🟡资源紧张"，他**立刻知道**：AI 还在努力但已经有点吃力了，可能需要我补充一些信息。这比 AI 默默挣扎然后突然说"建议手动"要好一万倍。

**📋 明约——"我们确认一下"**

交付前的人机双确认。AI 列出：目标匹配了吗？边界覆盖了吗？风险可控吗？用户说"交付"即确认。Clark 和 Brennan（1991）的"共同基础理论"[^29]告诉我们：有效协作的前提是双方对当前状态有**共享理解**。明约就是建立这个共享理解的仪式。

## 经验三域——AI 怎么"学以致用"

### 一个程序员的三种知识

你有没有想过，一个资深程序员脑子里的知识是怎么组织的？

我观察了很久，发现大概分三层：

1. **第一层：这个 bug 怎么修的** — 具体的、一次性的、和当前任务绑定的知识
2. **第二层：遇到 NullPointerException 应该先检查什么** — 跨越多次任务的、经过验证的模式
3. **第三层：永远不要相信外部输入** — 经过无数次验证的、接近"真理"的元规律

Benjamin Bloom（1956）的教育目标分类学[^33]其实说的是类似的事情：记忆→理解→应用→分析→评价→创造，层层递进。

PI 的经验三域就是按照这个逻辑设计的：

| 域 | 大白话 | 例子 | 生命周期 |
|---|-------|------|---------|
| ⚔️ **战域** | "这次学到了什么" | 这个项目用 Redis 做缓存比 Memcached 快 30% | 当前任务 |
| 🔮 **鉴域** | "跨项目也管用的经验" | 读密集型场景优先考虑缓存，写密集型场景慎用 | 跨任务 |
| 🏔️ **道域** | "刻在骨子里的铁律" | 没有银弹；过早优化是万恶之源 | 永久 |

### 升格五问：知识"升级"前的质量门

从低域升到高域，不是自动的。PI 要求回答五个问题：

1. **这是个例还是规律？** — 防止一次经验就过度泛化
2. **边界在哪？** — 防止适用范围模糊
3. **值不值得常驻？** — 防止认知过载
4. **和已有的规则矛盾吗？** — 防止自相矛盾
5. **跨 ≥3 次独立验证了吗？** — 防止偶然性

知识不是"学到就沉淀"，而是**"验证后才升格"**。这是科学方法论在知识管理中的应用。

### 为什么比传统 RAG 更好？

传统的 RAG（检索增强生成）把所有知识扔到一个大坑里，然后靠向量相似度检索[^37]。问题在于：没有质量控制，没有知识淘汰，检测不到矛盾。

PI 的经验三域提供了**结构化的知识治理**——什么是临时的，什么是永久的，什么需要反复验证。而"识"机制（自洽检测）则确保知识体系不会自相矛盾。

## 一座跨越两千年的桥

### 当孙子遇见博弈论

我在 PI 中大量引用了《孙子兵法》。这不是"为了显得有文化"——我越深入研究，越发现两千年前的兵家智慧和现代决策科学说的是**同一件事**：

| 孙子说 | 用现代话说 | PI 怎么用 |
|--------|----------|---------|
| 致人而不致于人 | 先发优势（game theory） | ⚡PI-04：主动掌控 |
| 知彼知己，百战不殆 | 完全信息博弈 | 启动三查：查境·查史·查标 |
| 兵无常势，水无常形 | 自适应策略 | 六阶战势动态切换 |
| 投之亡地然后存 | 承诺策略 | ⚡PI-02：穷理尽性 |
| 庙算多胜 | 期望效用最大化 | 方案比选矩阵 |

### 纵横家教我做交互设计

《鬼谷子》的捭阖之术[^39]给了 PI 人机交互的核心框架：

用户迷茫的时候——**捭（展开）**，打开选项让他选。这是 Norman（1988）的可见性原则[^40]。

用户目标明确的时候——**阖（收束）**，直接执行别废话。这是 Hick 定律[^41]：选项越少，决策越快。

用户情绪急切的时候——**先阖再捭**，先用一句话稳住情绪，再展开讨论。这是 Gross（1998）的情绪调节理论[^42]。

两千年前纵横家的说客术，和 2019 年 Microsoft Research 的人机交互指南，讲的是同一件事：**根据对方的状态，调整你的沟通策略。**

### 道家教我"少说废话"

道家的"上善若水"在 PI 中被翻译成了 ⚡PI-05：**好钢刀刃**——高信息密度，不说废话，深度思考后再输出。

这不是审美偏好。Davenport & Beck（2001）的《注意力经济学》[^43]指出：在信息过载的世界里，注意力是最稀缺的资源。AI 的每一次输出都在消耗用户的注意力。冗长的、重复的、没有信息增量的输出，是对用户认知资源的浪费。

"少说废话"不是态度问题，是**设计原则**。

## 所以，效果到底怎么样？

### AI 幻觉——最大的敌人

说了这么多理论，你一定想问：**效果呢？**

先说最严重的问题——AI 幻觉。大模型动不动就一本正经地胡说八道，这几乎是行业共识（Huang et al., 2023）[^1]。

PI 用了五道防线来对付它：

1. **搜→读→验→交付**（⚡PI-01）——强制验证环节，不允许跳过
2. **猜而不搜**（反模式十戒）——看到"应该是…""通常是…"就亮红灯
3. **明证格式**——每个结论必须附带假设+证据+排除项，使幻觉可被审计
4. **狼虎精神**——消除确认偏差，角色切换验证
5. **证据门**——杜绝空口无凭

### AI 最让人崩溃的习惯："建议你自己试试"

你有没有遇到过这样的 AI——你让它帮你修一个 bug，它尝试了两三次没搞定，然后说："建议你自己手动检查一下。"

**这是 AI 世界的"不作为"。** PI 用多层机制来治这个毛病：

- **⚡PI-02 穷理尽性**——方案没穷尽之前，禁止言退
- **六阶战势**——失败不是退缩的理由，而是升级策略的触发器
- **肃阵模式**——检测到退缩倾向时自动加强约束
- **善始善终协议**——即使最终真的解决不了，也要提供高质量的移交文档，而不是一句"建议手动"

### 调试效率的量变

传统 AI 调试方式：试错 → 猜测 → 试错 → 放弃。时间复杂度 O(n!)——无方向的穷举。

PI 的调试方式：读败 → 定界 → 溯源 → 比对 → 验假 → 固防。时间复杂度 O(n log n)——结构化搜索空间缩减。

区别在哪？**有没有方向感。** 传统方式像在迷宫里随机走，PI 像在迷宫里拿着地图走。

### 效果预期

我不会给你一个虚假的数字。但基于认知科学理论和框架设计原理，这是我的预期：

| 维度 | 传统 AI | 预期改善 | 靠什么 |
|------|--------|---------|-------|
| 幻觉率 | ~15-25% | <5% | 证据门+验证矩阵 |
| 任务完成率 | ~60-70% | >90% | 六阶战势+穷理尽性 |
| 调试平均轮次 | 5-8 轮 | 2-4 轮 | 调试六步+九令 |
| 人机误解率 | ~20-30% | <10% | 共振五式+交互三问 |

> 这些是理论预期，不是已验证的数据。真正的验证需要系统性的 A/B 测试——PI 项目的 benchmark/ 目录下正在开发这些工具。

## 未来的路

### 多 Agent 时代的认知协调

PI 已经有了 Leader-Teammate-Coach 的团队协作框架。但在多 Agent 协作越来越普遍的今天，还有很多需要探索的：

- **认知多样性**：为不同 Agent 配置不同的认知栈，就像组建一支多元化的团队
- **跨 Agent 知识共享**：一个 Agent 踩过的坑，另一个 Agent 不应该再踩
- **分布式元认知**：多个 Agent 如何共同感知"我们整体进展到哪了"

### 让知识学得更聪明

目前的升格协议使用固定阈值（验证 ≥3 次才升格）。未来可以更聪明——高频场景降低阈值（因为有更多验证机会），低频场景保持保守（防止过度泛化）。

### 跨会话的"记忆"

PI 当前最大的限制之一：每次新会话都从零开始。虽然有恢复协议（上下文快照+战域卡），但真正的跨会话认知连续性——让 AI 记住你、记住你的项目、记住你们之前合作的经验——这是下一个前沿。

### 基准测试与持续改进

任何框架都需要数据来验证效果。PI 的 benchmark 系统正在持续建设中——多维评分、跨模型对比、纵向跟踪。因为我们相信：**不能度量的进步不是进步。**

## 写在最后

回到最开始的那些日子——钉钉消息发不出去、卡片渲染不了、文件下载 403、AI 反复说"我知道了""我来修复"然后什么都没修好的日子。

那时候的我绝对没想到——这种愤怒和疲惫，最终会催生出一个融合了 Jung、Kahneman、孙子、鬼谷子的认知框架。

两周的密集思考和实践，加上异度神剑、MBTI、孙子兵法给我的灵感，PI 从一个"让 AI 别再说'建议手动'"的朴素愿望，长成了一整套**关于 AI 应该如何思考的理论**。

它将：
- **Jung 的认知功能**变成了 AI 的信息处理优先级
- **Kahneman 的双系统理论**变成了反模式十戒
- **Flavell 的元认知理论**变成了六阶战势
- **Bloom 的知识分层**变成了经验三域
- **Clark 的共同基础理论**变成了共振五式
- **东方古典智慧**提供了独特的策略直觉和表达力

而它最重要的特性是：**跨平台通用**。不管你用的是 Claude Code、Copilot CLI、Cursor 还是 Kiro，PI 都能用。因为它教的不是"某个平台的技巧"，而是"怎么思考"——这是通用的。

> **智者不惑，仁者不忧，勇者不惧。** ——《论语·子罕》

PI 的目标从来不是取代人类。它是要让人和 AI 真正成为**伙伴**——人类提供方向、判断和创造力，AI 提供速度、精度和穷尽式搜索。两者合在一起，远大于各自的总和。

这条路还很长。但每一次迭代、每一次失败、每一次顿悟，都在让这个框架变得更好。

因为正如六阶战势教给 AI 的那样：**失败不是终点，是升级的起点。**
